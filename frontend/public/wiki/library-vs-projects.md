# Library vs Projects

De applicatie werkt in twee lagen.

## Library

Hier definieer je de **baseline**:

- ETIM class
- features
- governed parameters
- mappings
- presets
- released versions

## Projects

Hier maak je concrete **instances** op basis van released typicals:

- parameterselecties
- projecttoevoegingen
- actieve drivers
- afgeleide interfaces

## Praktische regel

- `Library` zegt wat *mag*
- `Projects` zegt wat *effectief gekozen is*
